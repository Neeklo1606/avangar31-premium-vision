# Переключение админки на Vue (TrendAgent)

## Проблема
В админ-панели не было пункта меню **TrendAgent** (ЖК, Квартиры, Паркинги и т.д.), потому что `/admin` отдавала **старая** сборка из `dist/admin/` (отдельный Vue-проект в `backend/admin/`), а не Laravel + Vue SPA из `backend/resources/js/`.

## Решение
Обновлён **корневой `.htaccess`**:

1. **`/build/*`** — ассеты Vue-админки отдаются из `backend/public/build/` (сборка Vite в backend).
2. **`/admin` и `/admin/*`** — запросы уходят в **Laravel** (`backend/public/index.php`), который отдаёт шаблон `admin.blade.php` с подключением Vue SPA (TrendAgent).
3. **`/api/*`** — запросы к API тоже идут в Laravel (для работы админки с TrendAgent API).

В итоге при заходе на `/admin` выполняется Laravel, отдаётся страница с нашим Vue-приложением (меню TrendAgent, ЖК, Квартиры, Паркинги и т.д.).

## Что нужно на сервере
- В корне репозитория после деплоя должен быть актуальный **.htaccess** (из этого коммита).
- В **backend** должна быть выполнена сборка: `npm run build` (уже делается в `deploy.php`), чтобы существовала папка `backend/public/build/` с JS/CSS.

## Проверка
После деплоя откройте: **https://bot.siteaccess.ru/admin**  
Должны увидеть в сайдбаре заголовок **TrendAgent** и пункты: Главная, ЖК (Комплексы), Квартиры, Паркинги, Дома, Участки, Коммерция, Поселки, Проекты домов.
