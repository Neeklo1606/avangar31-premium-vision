# План: Hero / Zero-block — приведение к эталону

**Цель:** Привести главный экран (hero) на https://bot.siteaccess.ru/ к эталонному скрину 1.

**Файлы для изменения:**
- `src/components/sections/SearchSection.jsx` — основной hero-блок
- `src/components/layout/Layout.jsx` — обёртка main
- `src/components/ui/CategoryCard.jsx` — карточки категорий
- `src/components/ui/Tab.jsx` — переключатели (pills)
- `tailwind.config.js` — при необходимости (радиусы)
- `src/index.css` — глобальные стили (при необходимости)

## PROMPT 1 — HERO / ZERO-BLOCK: убрать пустоту и выровнять старт экрана

**Задача:** Контент начинается сразу после хедера, без лишнего воздуха сверху.

### Текущее состояние
- `SearchSection`: `pt-3 pb-4 lg:pt-4 lg:pb-6` на внутреннем контейнере
- `Layout`: `main` без padding, только `flex-1`
- Section: `w-full bg-white flex flex-col` — без лишних отступов

### Действия
1. **Layout.jsx**
   - Проверить: у `main` не должно быть `pt-*` или `mt-*`
   - Добавить класс `pt-0` явно, если где-то наследуется padding (опционально)

2. **SearchSection.jsx**
   - Уменьшить `pt-3` → `pt-2`, `lg:pt-4` → `lg:pt-3` (если ещё есть лишний воздух)
   - Убедиться, что между хедером и геолокацией нет дополнительных margin

3. **Проверка**
   - Открыть https://bot.siteaccess.ru/ на desktop (≥1024px)
   - Геолокация "Москва и МО" должна быть видна сразу под хедером
   - Hero не должен визуально "проваливаться" вниз

### Критерий готовности
- Первый экран плотный, без большого пустого пространства сверху
- Геолокация + заголовок видны без скролла (или с минимальным)

## PROMPT 2 — ГЕОЛОКАЦИЯ: отдельная кнопка, не часть hero

**Задача:** Геолокация — самостоятельная кнопка под хедером, слева.

### Текущее состояние
- Уже реализовано: `<button>` с иконкой pin, текстом "Москва и МО", стрелкой вниз
- Расположена в `flex justify-start` под хедером
- Полностью кликабельная, открывает RegionModal

### Действия
1. **SearchSection.jsx**
   - Визуально оформить как явную кнопку (не "ссылка-подпись"):
     - Добавить `rounded-lg`, `px-3 py-2`, `bg-transparent hover:bg-gray-50`
     - Сохранить: иконка pin слева, текст, стрелка вниз
   - Убедиться: нет `border-b`, `divider`, отдельной линии
   - Кнопка в пределах `max-w-container`, выровнена слева (`justify-start`)

2. **Структура**
   - Геолокация остаётся ВНЕ hero-блока (max-w-xl), т.е. на уровне основного контейнера
   - Hero-блок (заголовок, поиск, pills) — в `max-w-xl` по центру

### Критерий готовности
- Кнопка выглядит как отдельный UI-элемент выбора региона
- Без визуального разделителя (линии, секции)
- Расположена под хедером, слева

## PROMPT 3 — ЗАГОЛОВОК HERO: типографика и выравнивание

**Задача:** Заголовок по центру, "Live Grid." синий, остальное чёрный, компактные отступы.

### Текущее состояние
- `text-center` — уже есть
- "Live Grid." — `text-primary italic font-semibold`
- Остальной текст — `font-medium` (цвет `text-dark` от родителя)

### Действия
1. **SearchSection.jsx**
   - Заголовок: `className="text-center text-xl lg:text-2xl ..."`
   - Явно задать цвет второму span: `text-dark` (если не наследуется)
   - Уменьшить gap между заголовком и поиском:
     - Текущий `gap-4 lg:gap-5` в hero — возможно сократить до `gap-3 lg:gap-4`
   - Уменьшить отступ над заголовком (между геолокацией и заголовком):
     - Проверить `gap` в родительском flex — не добавлять лишний

2. **Типографика**
   - "Live Grid." — `text-primary italic font-semibold` ✓
   - "Более 100 000..." — `text-dark font-medium` ✓

### Критерий готовности
- Заголовок центрирован
- Цвета корректны, читается как главный акцент
- Нет слипания с поиском и нет лишнего воздуха

## PROMPT 4 — ПОИСКОВАЯ СТРОКА: одна линия (desktop)

**Задача:** [Input] [Кнопка фильтра] [CTA] в одну линию, 44–48px, ровные gap 8–12px.

### Текущее состояние
- Уже реализовано: Input (flex-1) + квадратная кнопка фильтра + CTA
- `flex flex-col sm:flex-row` — на mobile столбиком, на sm+ в ряд
- Высота 44px (h-11), gap-2 sm:gap-3

### Действия
1. **SearchSection.jsx**
   - Desktop (lg): гарантировать одну линию — `lg:flex-row` (сейчас sm:flex-row)
   - Высота: `h-11` (44px) или `h-12` (48px) — оставить 44px, соответствует эталону
   - Input: `bg-gray-50`, `border-gray-200/80`, `rounded-lg` ✓
   - Кнопка фильтра: `w-11 h-11`, `rounded-lg` ✓
   - CTA: `h-11`, `rounded-lg`, `px-5` ✓
   - Gap: `gap-2` (8px) или `gap-3` (12px) — проверить визуально

2. **Проверка**
   - На desktop вся строка в одну линию
   - Элементы не обрезаны, равномерные зазоры

### Критерий готовности
- Поиск — единый аккуратный UI-блок
- Все элементы одной высоты, ровные промежутки

## PROMPT 5 — ПЕРЕКЛЮЧАТЕЛИ КАТЕГОРИЙ (PILLS)

**Задача:** Pills 34–36px, одинаковые скругления, активная синяя, неактивные светлые, ряд по центру hero.

### Текущее состояние
- Tab с `variant="hero"`, `size="hero"` (h-9 = 36px)
- TabGroup `layout="hero"`: `flex justify-center gap-2`
- Активная: `bg-primary text-white rounded-lg`
- Неактивная: `bg-gray-50 border border-gray-light text-gray-medium rounded-lg`

### Действия
1. **Tab.jsx**
   - Проверить высоту: `h-9` = 36px ✓
   - Убедиться: неактивные без визуального шума (без тени, без жирной рамки)
   - Одинаковые скругления: `rounded-lg` для всех ✓

2. **SearchSection.jsx**
   - TabGroup `layout="hero"` — центр hero ✓
   - Gap между pills: `gap-2` (8px) — при необходимости `gap-2.5` (10px)

### Критерий готовности
- Pills аккуратные, ряд центрирован
- Активная — синяя с белым, неактивные — светлые

## PROMPT 6 — СЕТКА КАРТОЧЕК КАТЕГОРИЙ: размеры и пропорции

**Задача:** Карточки крупнее, 2 ряда, больший gap, светло-серый фон, скругление 16–20px.

### Текущее состояние
- `grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-7` — 7 колонок на desktop
- `gap-1.5 lg:gap-2` — мелкий gap
- CategoryCard: `bg-gray-100`, `rounded-lg` (12px), `min-h-[72px]`

### Действия
1. **SearchSection.jsx**
   - Сетка: пересмотреть раскладку под "2 визуальных ряда"
     - Вариант: `grid-cols-4 lg:grid-cols-5` — 2 ряда по 4–5 карточек
     - Или: оставить 7 колонок, но увеличить высоту и gap
   - Gap: `gap-3 lg:gap-4` (12–16px)

2. **CategoryCard.jsx**
   - Фон: `bg-gray-50` или `bg-gray-100` (светло-серый) ✓
   - Скругление: `rounded-xl` (16px) или `rounded-2xl` (20px)
   - Высота: увеличить `min-h-[72px]` → `min-h-[88px]` или `min-h-[96px]`

### Критерий готовности
- Карточки выглядят "воздушными" и крупными
- Скругление 16–20px, увеличенный gap

## PROMPT 7 — ВНУТРЕННИЙ LAYOUT КАРТОЧКИ

**Задача:** Текст слева сверху, иконка справа, иконки крупнее, padding 16–20px, object-fit: contain.

### Текущее состояние
- Layout: текст слева, иконка справа ✓
- Иконка: `w-14 h-14 lg:w-16 lg:h-16` (56–64px)
- Padding: `p-3 lg:p-4` (12–16px)
- `object-contain` ✓

### Действия
1. **CategoryCard.jsx**
   - Текст: закрепить слева сверху — `items-start` для flex, `justify-start`
   - Padding: `p-4 lg:p-5` (16–20px)
   - Иконка: увеличить до `w-16 h-16 lg:w-20 lg:h-20` (64–80px)
   - `object-fit: contain` — уже есть через `object-contain`
   - Убедиться: `overflow-hidden` не обрезает иконку (object-contain внутри контейнера)

### Критерий готовности
- Текст слева сверху, иконка справа
- Иконки крупнее, без обрезаний
- Внутренние отступы 16–20px

## PROMPT 8 — MOBILE-FIRST: приоритет мобильной версии

**Задача:** Hero на mobile — вертикальный порядок, поиск и CTA на всю ширину, 2 карточки в ряд, нормальные отступы.

### Текущее состояние
- Порядок: геолокация → заголовок → поиск (+ filter + CTA) → pills → категории ✓
- Поиск: `flex-col sm:flex-row` — на mobile столбиком
- CTA: `w-full sm:w-auto` — на mobile full width ✓
- Карточки: `grid-cols-2` — 2 в ряд ✓

### Действия
1. **SearchSection.jsx**
   - Проверить порядок на mobile (≤640px): геолокация → заголовок → поиск → pills → категории
   - Поиск: Input и CTA на всю ширину на mobile ✓
   - Filter button и CTA: не должны упираться — `gap-2` между ними ✓
   - Карточки: `grid-cols-2` с `gap-3` (не `gap-1.5`) для нормального отступа
   - Padding контейнера: `px-4` — не прилипает к краям ✓

2. **Layout**
   - Hero-блок: на mobile `max-w-xl` может сужать — убедиться, что `px-4` даёт отступы от краёв

3. **Тестирование**
   - Viewport 375px, 390px, 414px
   - Всё читаемо, кнопки не перекрываются
   - Карточки 2 в ряд, с gap

### Критерий готовности
- Mobile выглядит современно и удобно
- Уровень ЦИАН по UX
